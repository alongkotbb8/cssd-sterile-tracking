## ขอบเขตงานรอบนี้ = เฟส 1 (MVP) เท่านั้น
สร้างให้เดินได้ end-to-end ตามลำดับนี้ อย่าข้ามไปเฟส 2/3:
1. โครง monorepo ตาม CLAUDE.md + README + .env.example + docker-compose สำหรับ PostgreSQL
2. โมเดลข้อมูล + Prisma schema + migration + seed ข้อมูลตัวอย่าง (แผนก: ทันตกรรม/ER/ห้องคลอด/ทำแผล, set template ตัวอย่าง)
3. Auth + role-based (CSSD / SUPERVISOR / ADMIN) + AuditLog middleware
4. API: สร้างชุด + ออกเลขรันจาก backend + endpoint สั่งพิมพ์ (ผ่าน MockPrinterAdapter ที่ออก ZPL ลง log/ไฟล์)
5. API: บันทึก movement เข้า–ออก (เลือกแผนกปลายทาง, ผู้รับ optional) + กฎบล็อกของหมดอายุ + คำนวณ expiry อัตโนมัติ
6. Mobile (Flutter): สแกน QR, นำเข้า/นำออก (batch scan), แดชบอร์ดกราฟโดนัท (คงเหลือตามชนิดชุด + สัดส่วนตามแผนก), หน้าออกรายงานรายสัปดาห์เลือกช่วงวันที่
7. แจ้งเตือน: ใกล้หมดอายุ + ลืมสรุปรายวัน (เริ่มที่ scheduled job ฝั่ง server + โครง FCM)
