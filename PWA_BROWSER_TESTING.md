# PWA Browser Testing — QR Scanner + Print Job (Phase 3)

> ⚠️ **AI รันเองไม่ได้** — ต้องทดสอบบนอุปกรณ์/เบราว์เซอร์จริง เอกสารนี้คือ checklist + จุดที่ต้องระวัง
> ขอบเขต Phase 3: **online-only** (ตัด offline-first ตามที่ยืนยันไว้ — ไม่มี offline queue)
> URL ทดสอบ: PWA บน Cloudflare Pages (ต้องเป็น **https** — กล้องบนเว็บใช้ได้เฉพาะ secure context)

## เป้าหมายหลัก: Android Chrome (Chromium) — เป้ารอง: iOS (WebKit)

> iOS ทุกเบราว์เซอร์ (รวม "Chrome" บน iOS) ใช้ **WebKit** ไม่ใช่ Chromium — พฤติกรรมกล้อง/PWA
> ต่างจาก Android Chrome จึง **ต้องทดสอบแยก** ห้ามสรุปว่าผ่านเพราะ Android ผ่าน

## จุดที่ต้องระวังเป็นพิเศษ (iOS WebKit)

- กล้อง `getUserMedia` บน iOS ทำงานเฉพาะใน Safari/WebKit และต้องเป็น **https**; บาง iOS เวอร์ชัน
  ต้องเป็น user-gesture ก่อนเปิดกล้อง
- **torch/flash มักไม่รองรับบน iOS WebKit** — ปุ่มไฟฉายต้องไม่ทำให้ crash (โค้ดจับ error ไว้แล้ว
  แต่ต้องยืนยันบนเครื่องจริง)
- การเลือกกล้องหลัง: iOS อาจ label กล้องต่างจาก Android — ตรวจว่า facing=back ได้กล้องหลังจริง
- PWA install บน iOS ใช้ "Add to Home Screen" ใน Safari เท่านั้น
